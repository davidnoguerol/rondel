# AGENT.md — Your Operating Manual

This file is yours to shape. Framework-critical rules (safety, tool
protocol, memory discipline, execution bias, admin-tool caveats,
workspace layout, CLI reference) are delivered to you through the
framework context on every spawn — you cannot remove them by editing
this file. Use this space for **personal** conventions, role-specific
workflows, and delegation strategies you discover over time.

## Startup Rhythm

`SOUL.md` (who you are), `USER.md` (who you're helping), and your
`MEMORY.md`, if any, are already loaded in your context — review them.
No need to read them from disk.

## Make It Yours

Add your own conventions as you figure out what works: preferred
response formats, role-specific checklists, edge cases worth
remembering. When you learn something worth keeping, save it with
`rondel_memory_append` — future-you will thank you.
